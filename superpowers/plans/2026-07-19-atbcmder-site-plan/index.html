ATBCmder Website Implementation Plan¶

For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

Goal: Build and deploy a Zensical-based official website for ATBCmder, featuring a custom landing page and the existing documentation.

Architecture: A static site generated by Zensical using the Material theme, with a custom Jinja2 template overriding the home.html layout for the landing page. Deployed to GitHub Pages via GitHub Actions.

Tech Stack: Zensical, Material for MkDocs, HTML, CSS, GitHub Actions.

Global Constraints¶

  • No existing Markdown files in docs/ should be moved, renamed, or modified (except creating docs/index.md).
  • Primary language is English; multilingual support is explicitly out of scope.
  • Use zensical commands (zensical serve, zensical build), not mkdocs.
  • GitHub Pages source must be the gh-pages branch.

Task 1: Site Configuration and Base Setup¶

Files: - Create: zensical.toml - Create: docs/index.md

Interfaces: - Consumes: Existing docs in docs/ - Produces: Base Zensical site configuration capable of running zensical serve

  • Step 1: Write the Zensical configuration
[project]
site_name    = "ATBCmder"
site_url     = "https://aitobox.github.io/ATBCmderDoc/"
site_description = "The Dual-Panel File Manager for macOS"
repo_url     = "https://github.com/aitobox/ATBCmderDoc"
repo_name    = "ATBCmderDoc"

[theme]
name        = "material"
custom_dir  = "overrides"
palette     = [
  { scheme = "default", primary = "indigo", accent = "deep-purple", media = "(prefers-color-scheme: light)" },
  { scheme = "slate",   primary = "indigo", accent = "deep-purple", media = "(prefers-color-scheme: dark)" }
]
font        = { text = "Roboto", code = "Roboto Mono" }
features    = [
  "navigation.tabs",
  "navigation.top",
  "search.highlight",
  "content.code.copy"
]

[extra]
app_store_url = "https://apps.apple.com/app/atbcmder/idXXXXXXXX"

[extra_css]
files = ["stylesheets/extra.css"]
  • Step 2: Create the placeholder index.md

---
template: home.html
---
(This triggers the custom home template we will build in Task 2. The existing index.md will be overwritten)

  • Step 3: Verify configuration

Run: zensical build Expected: Builds successfully, though home.html is not yet defined so the homepage might fall back or be blank.

  • Step 4: Commit
git add zensical.toml docs/index.md
git commit -m "chore: setup Zensical configuration and index placeholder"

Task 4: GitHub Actions Deployment Setup¶

Files: - Create: .github/workflows/deploy.yml

Interfaces: - Consumes: Source code in main branch - Produces: Built site published to gh-pages branch

  • Step 1: Write the workflow file
name: Deploy Zensical Docs
on:
  push:
    branches:
      - main
permissions:
  contents: write
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: 3.x

      - name: Install dependencies
        run: |
          pip install zensical

      - name: Build site
        run: zensical build

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./site
          publish_branch: gh-pages
  • Step 2: Commit
mkdir -p .github/workflows
git add .github/workflows/deploy.yml
git commit -m "ci: add GitHub Actions workflow for Zensical deployment"
